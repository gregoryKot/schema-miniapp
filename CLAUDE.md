# Правила фронтенда

## Архитектура

- React + TypeScript, Vite, Telegram Web App SDK
- Нет роутера — видимость экранов через boolean state в `App.tsx`
- Все API-вызовы только через `src/api.ts`
- Стили — инлайн `style={{}}` в JSX. Нет CSS-модулей, нет styled-components.

## Цвета — ОБЯЗАТЕЛЬНЫЕ правила

Все цвета должны идти через CSS-переменные из `src/index.css`. Никогда не пиши hex или rgba напрямую для текста.

### Цвет текста

| Что | Переменная |
|-----|-----------|
| Основной текст | `var(--text)` |
| Вторичный (подписи, описания) | `var(--text-sub)` |
| Бледный (hint, placeholder) | `var(--text-faint)` |
| Текст на выбранных chip | `var(--chip-sel-text)` |

### Акцентные цвета (текст и иконки)

| Цвет | Переменная |
|------|-----------|
| Фиолетовый (основной) | `var(--accent)` |
| Индиго | `var(--accent-indigo)` |
| Оранжевый | `var(--accent-orange)` |
| Синий | `var(--accent-blue)` |
| Зелёный | `var(--accent-green)` |
| Жёлтый | `var(--accent-yellow)` |
| Красный | `var(--accent-red)` |

### Фоны и рамки (можно использовать rgba)

Для фонов и border — `rgba(var(--fg-rgb),0.X)` — допустимо, адаптируется к теме:
- Лёгкий фон карточки: `rgba(var(--fg-rgb),0.04)`
- Разделитель: `rgba(var(--fg-rgb),0.07)`
- Рамка: `rgba(var(--fg-rgb),0.1)`

### Запрещено

```tsx
// НЕЛЬЗЯ — сломается в светлой теме:
color: '#a78bfa'
color: 'rgba(167,139,250,0.6)'
color: 'rgba(var(--fg-rgb),0.4)'  // <0.66 — неразборчиво на светлом фоне
```

```tsx
// МОЖНО:
color: 'var(--text-sub)'
color: 'var(--accent)'
background: 'rgba(var(--fg-rgb),0.05)'  // для фонов OK
```

## Смена темы

Вся тема — в одном файле `src/index.css`. Тёмная тема: `:root`, светлая: `html[data-theme="light"]`. Чтобы изменить цвет во всём приложении — меняй переменную в этом файле.

## Правила кода

- Максимум ~200 строк на компонент. Если больше — выносить логику в хук или хелпер.
- Не дублировать API-вызовы — только через `src/api.ts`.
- Оптимистичные обновления: после create/delete обновлять локальный state напрямую, не рефетчить всё.
